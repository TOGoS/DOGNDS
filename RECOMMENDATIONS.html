DOGNDS Implementation Recommendations

Optional standards to follow for implementation details of various subsystems.

Simple Data Repository

Indexes data using the filesystem.

  • .../data/<sector>/<first 2 characters of base32-encoded SHA1>/<base32-encoded SHA1>

Where <sector> is a name (conventionally all lowercase with dashes to separate multiple words) indicating the role of the data stored there. Many repositories will have only a single data sector, but this structure allows multiple sectors to ease management of data from different sources.
